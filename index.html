Why Do We Need a Playbook?

Everyone at STAGE uses AI. But without a shared setup...

⏳ Everyone wastes time setting up

New person joins → spends hours figuring out which tools to install, asking around on Slack, copying random configs from teammates.

🔀 Everyone has a different setup

Person A has different AI skills than Person B. What works for one doesn't work for another. No consistency.

🚫 No safety guardrails

AI writes code that leaks API keys, pushes directly to main, skips tests. No one catches it until production breaks.

💭 Knowledge stays in people's heads

"Ask Rahul how to set up Flutter" or "Divya knows the backend setup." When they're busy, everyone waits.

The Playbook fixes all of this. One command → same tools, same rules, same superpowers for everyone.

Built-In Safety Guardrails

The playbook teaches Claude what to NEVER do. So you can vibe-code safely.

❌ Without Playbook

Claude might push code directly to main branch
Claude might commit your API keys to GitHub
Claude might say "done" without actually testing
Claude might break other people's code
Claude might skip security checks

✅ With Playbook

Claude always creates a separate branch
Claude never commits secrets or API keys
Claude must run tests before saying "done"
Claude reads existing code before changing it
Claude follows OWASP security rules

💡   These rules are invisible to you. Once the playbook is installed, Claude follows them automatically. You don't have to remember anything.
